🛠 Технологический стек — простыми словами

Интерфейс (то, что видит пользователь)
Next.js + React + TypeScript
Современный веб-фреймворк. Быстрая загрузка страниц, работает и на телефоне, и на компьютере. Язык программирования TypeScript защищает от ошибок.
Next.js 15 React Tailwind CSS TypeScript
Серверная логика (невидимая часть)
Server Actions + Prisma ORM
Авторизация, работа с базой данных, бизнес-логика. Prisma — инструмент для безопасной работы с базой без риска ошибок.
Node.js Prisma bcrypt JWT
База данных
SQLite / LibSQL (Turso)
Лёгкая и надёжная база данных. Для MVP и демонстрации — идеальный выбор: не требует отдельного сервера, легко масштабируется при росте.
SQLite LibSQL Turso-ready
Искусственный интеллект
OpenAI-совместимый API
Платформа подключена к любой LLM-модели через единый интерфейс (Claude, GPT и др.). Системный промпт на русском — ИИ знает специфику закупок.
OpenAI API Claude Prompt Engineering

📁 Структура проекта — что где лежит

prokadry/
app/ — страницы сайта
page.tsx — главная (лендинг)
auth/page.tsx — вход и регистрация
dashboard/page.tsx — личный кабинет
api/chat/route.ts — ИИ-ассистент (API)
actions/auth.ts — логика входа и регистрации
components/ — визуальные блоки
landing.tsx — главная страница
layout.tsx — меню и оболочка кабинета
employer.tsx — все разделы работодателя
seeker.tsx — все разделы соискателя
admin.tsx — администраторская панель
registry.tsx — реестр резюме с поиском
resume.tsx — карточка кандидата
regions.tsx — региональный модуль
ui.tsx — библиотека кнопок, значков, карточек
lib/ — ядро системы
types.ts — описание всех объектов (резюме, вакансия, пользователь…)
db.ts — подключение к базе данных
session.ts — управление сессиями пользователей
mock-data.ts — 75 резюме для демонстрации
Весь код в одном репозитории, одна команда для запуска. Заказчик получает полный исходный код при подписании акта (п. 8.1 договора).

🎤 Как рассказать Заказчику — сценарий презентации

1
Начать с лендинга (prokadry.ru)
«Вот что видит любой посетитель — кто мы, что предлагаем, для кого. Отсюда два пути: войти или зарегистрироваться.»
2
Зайти как Работодатель
«Вот кабинет работодателя. Здесь он ищет специалистов по закупкам — фильтрует по региону, опыту, 44-ФЗ или 223-ФЗ, смотрит карточку, отправляет приглашение.»
3
Переключиться на Соискателя
«А вот как видит платформу специалист по закупкам — его резюме, входящие предложения, переписка с работодателем.»
4
Показать Администратора
«Это ваш инструмент управления — вы видите всех пользователей, модерируете резюме, верифицируете компании, смотрите журнал действий.»
5
Упомянуть ИИ
«Платформа уже подключена к ИИ. Он знает о 44-ФЗ, 223-ФЗ, умеет объяснять разделы и помогать пользователям. В следующем этапе ИИ появится прямо в интерфейсе.»
6
Акцент на технологии
«Сделано на Next.js — том же фреймворке, что использует Vercel, TikTok, Notion. Быстро, безопасно, легко масштабировать. Вы получаете полный исходный код.»
Важно для показа: заранее войти под каждой ролью в разных вкладках браузера — переключение будет мгновенным и произведёт впечатление.
